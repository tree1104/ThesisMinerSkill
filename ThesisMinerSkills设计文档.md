# ThesisArchitect SKILL 设计架构文档
**版本**：v2.0 (Skill-Native & I/O Agnostic Edition)
**设计理念**：无状态执行 · I/O 多态自适应 · 四维创意引擎 · 硬约束自动修复 · 开题直出

---

## 1. SKILL 定位与边界

### 1.1 核心定位
ThesisArchitect 是一个专用于研究生开题阶段的高阶智能体技能。它作为底层大模型运行时（如 Claude Code）的即插即用插件，通过内建的学术谱系解析、多维创意激发与硬性学术约束机制，生成符合真实学术生存法则的实战论题，并一键直出开题报告核心素材。

### 1.2 职责边界（严格隔离）
- **SKILL 负责（核心功能）**：I/O 意图识别与多态路由、非结构化谱系解析、四维策略创意涌现、学术规范硬约束校验与自动修复、开题报告结构化生成。
- **SKILL 不负责（交由运行时）**：不维护多轮对话状态、不管理上下文窗口与 KV 缓存、不记录 Token 账单。这些全部由底层 Agent 运行时原生托管。

## 2. I/O 多态自适应设计

为提供极致的用户体验，SKILL 摒弃死板的文件强制依赖，采用**意图识别与多态路由**机制，支持对话与文件双通道输入输出。

### 2.1 输入路由
SKILL 激活后，按以下优先级解析输入上下文：
1. **文件输入 (`/input/` 目录)**：若用户指令中包含 `@input/` 或 SKILL 检测到 `/input/profile.json` 与 `/input/lineage.md`，则优先读取文件内容作为基础上下文。
2. **对话输入**：若未提供文件，SKILL 通过对话引导获取必要信息。用户可直接在对话框输入导师项目名称、同门论文摘要及个人学位信息。
3. **混合输入**：支持以文件为背景底座，在对话中追加临时指令（如：“读取input里的信息，但我现在想换个方向做大模型”）。

### 2.2 输出路由
SKILL 根据用户指令中的动词与修饰语，自动决定输出通道：
1. **文件输出**：当用户指令包含“保存”、“生成文件”、“输出到本地”等意图时，SKILL 将结果写入 `/output/` 目录（生成 `proposals_<timestamp>.json` 及 `draft_<timestamp>.md`），并在对话框回复文件路径摘要。
2. **对话输出**：默认模式。将结构化提案与开题草稿以高可读性的 Markdown 格式直接打印在对话框中，方便用户即时阅读与复制。

## 3. 核心组件强化设计

SKILL 内部由四个高内聚的纯逻辑模块组成，接收输入元数据，输出结构化结果。

### 3.1 谱系解析器
- **功能强化**：提升对非结构化文本的鲁棒性解析。不仅提取实体，更挖掘“未走完的路”。
- **处理逻辑**：
  - 实体抽取：识别“导师项目”（提取名称、核心目标）、“同门论文”（提取标题、使用方法、局限性）。
  - 边缘探测：标记同门论文中提及的“未来工作”或“受限于算力/数据未展开”的表述，作为高价值切入点。

### 3.2 四维创意涌现引擎
- **功能强化**：从三个策略扩展为四维模型，增加“矛盾驱动”，并引入候选自评分机制。
- **策略定义**：
  1. **导师项目延伸**：将大项目拆解为可在规定学制内完成的子课题（如：将“医疗大模型研发”拆解为“特定科室的问询微调”）。
  2. **同门成果继承**：基于边缘探测的局限点，引入新变量或迁移至新场景。
  3. **跨域联想**：当识别到多个不相关学科概念时，生成“A领域方法解B领域问题”的候选。
  4. **矛盾驱动挖掘**：检测“现有方法能力边界”与“实际需求”的语义矛盾（如：需要高精度但现有方法在噪声下失效），基于矛盾生成论题。
- **自评分机制**：每个候选生成后，内部根据“可行性(40%) + 创新度(30%) + 谱系贴合度(30%)”打分，过滤掉低于 6 分（满分10分）的方向。

### 3.3 硬约束校验与自动修复器
- **功能强化**：不仅做拦截，更提供确定性的自动修复能力，避免大模型反复重试浪费资源。
- **约束与修复规则**：
  - **标题格式**：≤20字；不以“研究/分析/探讨/设计”等主动动词开头；不匹配“基于X的Y研究”模式。
    - *自动修复*：超长标题执行依存句法截取核心名词短语；动词前置结构自动转换为名词性短语（如“研究X”→“X的研究”）。
  - **学术日历**：校验研究周期，硕士≤12个月，博士≤24个月。超期不熔断，但自动在研究内容中注入“分阶段并行执行”的降级策略提示。
  - **文献基线**：要求生成的综述大纲至少规划硕士 30 篇/博士 50 篇文献的检索方向。
  - **逻辑自洽**：校验“研究内容”与“研究目标”是否存在语义重合度 > 70% 的情况，若存在则标记 `WARNING: 内容与目标重合度过高`。

### 3.4 开题报告直出器
- **功能强化**：严格对齐国内双一流高校标准开题模板，强制大模型按结构填充。
- **模板控制**：内置“选题依据、国内外研究现状、研究内容与关键问题、研究方案与可行性、进度安排”五大模块。通过 Prompt 注入“必须包含拟解决的关键科学问题”等硬性指令。

## 4. SKILL 执行工作流

当运行时调用 `ThesisArchitect` SKILL 时，按以下无状态步骤执行：

1. **I/O 意图识别**：解析用户指令，确定输入源（文件/对话/混合）与输出目标（文件/对话）。
2. **上下文加载与解析**：读取并解析学位、学科、导师项目与同门基础信息，构建内部 `LineageGraph`。
3. **四维创意发散**：并行调用四个策略生成原始候选，执行自评分，保留 Top 3-5 个方向。
4. **结构化精炼**：针对每个候选，调用大模型填充 `title`, `problem_awareness`, `research_significance`, `differentiation`, `research_content` 等字段。
5. **硬约束拦截与修复**：通过本地校验器修复标题格式，标记不可行方案，注入文献基线要求。
6. **多态输出**：
   - 若选定为对话输出，将结果渲染为 Markdown 打印。
   - 若选定为文件输出，将结果序列化为 JSON 与 Markdown 写入 `/output/` 目录。

## 5. Prompt 规范与约束定义

为保证大模型输出质量，SKILL 内部调用大模型时强制注入以下约束模板：

```text
[SYSTEM]
你是一个严谨的学术导师。必须遵守：
1. 标题不超过20字，无主动动词开头，杜绝"基于X的Y研究"套路。
2. 必须基于提供的导师项目或同门论文谱系生成，严禁凭空捏造。
3. 输出必须严格符合要求的 JSON Schema。

[USER]
基于以下谱系信息：
{lineage_graph}

请使用 {strategy} 策略，生成 1 个 {degree} 级别的论题提案。
必须包含字段：title, problem_awareness, research_significance, differentiation, research_content, literature_review_outline。
```

## 6. 开题报告直出模板设计

当用户指令包含“生成开题报告”或“直出草稿”时，SKILL 提取最优提案，注入以下模板：

```markdown
# 开题报告：[提案标题]

## 一、选题依据与研究意义
### 1.1 现实/理论痛点
[problem_awareness]
### 1.2 研究意义（理论+实际）
[research_significance]

## 二、国内外研究现状
[literature_review_outline，要求坦诚陈述已有工作并指明不足，规划文献数量满足基线]

## 三、研究内容与关键问题
### 3.1 研究内容
[research_content]
### 3.2 拟解决的关键科学问题
[基于 differentiation 提炼的 1-2 个核心科学问题]

## 四、研究方案与可行性分析
### 4.1 研究方法与技术路线
[基于 research_content 生成的具体方法步骤]
### 4.2 可行性分析
[周期可行性、数据/设备可行性论证]

## 五、进度安排
[基于 academic_calendar 约束生成的甘特图节点列表]
```

## 7. 部署与调用示例

### 7.1 部署
将 `thesis_architect` 目录放置于运行时（如 Claude Code）的 `/skills/` 目录下。运行时将在启动时自动加载该 SKILL 的指令与工具定义。

### 7.2 调用示例（覆盖 I/O 多态场景）

**场景 1：纯对话输入输出（快速发散）**
> 用户："我是硕士生，导师在做医疗大模型，同门做的是问诊微调。帮我生成3个论题。"
> SKILL 动作：解析对话提取谱系 -> 执行四维创意 -> 对话框直接输出 3 个结构化提案。

**场景 2：文件输入 + 对话输出（详尽开题）**
> 用户："读取 @input/profile.json 和 @input/lineage.md，基于这些信息帮我生成开题报告草稿，直接发给我。"
> SKILL 动作：读取文件解析谱系 -> 生成提案并选最优 -> 对话框直出完整 Markdown 开题报告。

**场景 3：混合输入 + 文件输出（沉淀落盘）**
> 用户："读取 @input/ 里的背景，但我现在想结合最新的 RAG 技术做跨域联想。生成5个提案，保存到本地。"
> SKILL 动作：读取文件底座 + 追加 RAG 跨域指令 -> 生成提案 -> 写入 `/output/proposals_20260619.json`，回复“已保存至该路径”。
