# ThesisMinerSkill V4.0 架构设计文档
**—— 从“开题生成器”到“全生命周期学术导航员”**

**版本**：V4.0 (Full-Journey Navigator Edition)
**架构基线**：ThesisMinerSkill V2.1（三层物理结构）
**设计理念**：交互节制 · 信息确权 · 深度陪跑 · 降重脱敏 · 去AI痕迹

---

## 1. 演进背景与核心目标

基于 V2.1 架构在真实学术场景下的穿透性诊断，V4.0 致力于解决以下核心痛点：
1. **全流程引导线过短**：生成开题报告后智能体“静默”，未接管后续的文献精读与实验设计。
2. **信息获取草率**：初步获取信息后直接产出论题，缺乏时效性佐证与新颖性验证。
3. **输出模式粗糙**：缺乏颗粒度控制，无法适应不同学科对开题报告详细程度的迥异需求。
4. **AI 痕迹过重**：模板化输出导致“首先、其次、综上所述”等 LLM 典型句式频发，查重率与 AI 检测率双高。

V4.0 将原 V2.1 的线性六步流重构为 **“五阶段闭环导航流”**，在维持三层物理结构与去上下文化 I/O 哲学的基础上，植入学术常识校验、用户交互控制感与长期主义价值。

---

## 2. 五阶段闭环导航流（核心工作流重构）

V4.0 的核心业务流程从“单次生成”升级为“全生命周期陪跑”，强制引入中断点与确权门禁。

### 阶段一：信息确权与双时域联网勘探（解决建议 2, 3, 4）
废除“信息获取 -> 直出论题”的跳跃逻辑，引入**信息确权门禁**。

1. **开题灵感勘探（默认近 2 年）**：
   - 解析用户初始输入后，**不直接生成论题**。
   - 自动生成 3~5 组高精度学术检索式（含布尔运算符），调用联网搜索 API。
   - **强制中断**：向用户展示检索结果摘要（Top 5 篇最新文献），提供“时间范围拨盘”（提示：“当前默认展示近 2 年前沿动态，可调整为 1 年/3 年”）。
   - 用户确认“信息已阅”后，解锁进入四维创意引擎。
2. **候选论题重复度评估（默认近 5 年）**：
   - 四维引擎生成 Top 5 候选后，**必须进入重复度评估阶段**。
   - 自动基于候选标题生成查重检索式，联网检索近 5 年（可交互调节）的硕博论文与期刊。
   - 输出“新颖性风险评估雷达图”（文字版），明确标注重合度百分比及差异化空档。

### 阶段二：谱系解析与四维创意生成（复用 V2.1 核心）
- `lineage_parser` + `idea_generator` 执行，但**强制注入阶段一检索到的前沿热点作为“跨域联想”与“矛盾驱动”的种子语料**。
- 输出附带自评分与创新点说明的候选论题。

### 阶段三：多粒度开题生成与降重脱敏（解决建议 5, 7）
1. **多维度可控生成**：
   - 强制询问用户：“您需要【精简】、【标准】还是【详实】版的开题报告草稿？”
   - **精简版**：五大模块骨架，仅保留核心结论。
   - **标准版**：五大模块 + 每个研究内容配 1 个技术路线子图描述。
   - **详实版**：标准版 + 现状分条详细评述 + 风险矩阵 + 预期成果价值阐述。
2. **学术风格中性化**：
   - `report_generator` 渲染后，必须经过 `style_normalizer` 处理。
   - 执行禁用词表过滤、主动被动态互换、文献综述“问题-方法-不足”矩阵改写。
   - 输出 Markdown + 局部高重复风险段落标红提醒。

### 阶段四：用户决策与硬约束修复
- 【用户决策】：用户从带查重评估的候选中选择最终 1 个论题进入报告生成。
- `constraint_checker` 执行标题/逻辑/周期自动修复。

### 阶段五：深度辅助与实验映射闭环（解决建议 1, 6）
报告输出后，**禁止结束对话**。系统自动附加“深度辅助导航菜单”，将辅助从纯文本建议升级为**可执行的工单**。

1. **文献精读工作簿生成**：
   - 针对报告中的“国内外研究现状”，自动生成 3 篇标志性文献的精读框架。
   - 包含：三明治拆解（动机-方法-局限）、GAP 分析（与自身课题关键问题的映射）、可借鉴图表。
2. **实验/应用预研映射**：
   - **理工/计算机类**：检测到数学建模/代码仿真需求，输出“最小可行性实验（MVE）清单”（库版本、核心算法伪代码、基线对比计划）。
   - **数学/理论类**：检测到定理推演需求，输出“证明路径图谱”（前置引理梳理、核心引理构造思路、数值算例验证方案）。
   - **社科/商科类**：检测到实证调研需求，输出“调研方案设计”（问卷量表映射、抽样策略、数据分析模型预设）。
3. **逻辑压力测试（答辩模拟）**：
   - AI 扮演严苛盲审专家，针对“关键问题”发起苏格拉底式诘问，进行 3 轮答辩模拟追问。

## 3. 三层物理结构与资源层增量设计

V4.0 继续沿用 V2.1 的三层物理结构（元数据层 / 指令层 / 资源层），重点对资源层进行深度扩充。

### 3.1 资源层增量更新表

| 路径 | 新增/修改文件 | 职责说明 |
| --- | --- | --- |
| `core/references/` | `search_strategies.json` | **新增**。定义不同阶段的检索式生成模板、默认时间窗口（灵感期 2 年 / 重复度期 5 年）、可调节步长。 |
| `core/references/` | `forbidden_ai_phrases.json` | **新增**。收录 200+ 高频 AI 化术语及学术中性映射表（如 "显著提升" -> "呈现正向关联"）。 |
| `core/references/` | `output_granularity.yaml` | **新增**。定义“精简/标准/详实”三级颗粒度对应的 Markdown 标题深度、列表层级、字数阈值。 |
| `core/references/` | `prompt_templates.json` | **修改**。开题报告模板将流水账结构替换为“批判性矩阵”引导词。 |
| `core/scripts/` | `idea_generator.py` | **修改**。`generate_ideas` 增加 `search_feeds` 参数，强制注入联网检索热点作为种子语料。 |
| `core/scripts/` | `constraint_checker.py` | **修改**。增加 `check_novelty()` 方法，输出联网查重重合度与风险评级。 |
| `core/scripts/` | `report_generator.py` | **修改**。`generate_report` 增加 `granularity` 与 `style_neutral` 参数，动态渲染不同深度模板。 |
| `core/scripts/` | `style_normalizer.py` | **新增**。包含 `remove_ai_traces(text)` 方法：词频替换、依存句法重构、主动被动态互换。 |
| `core/scripts/` | `deep_helper.py` | **新增**。包含 `literature_deep_reader()`、`experiment_designer()`、`thesis_defense_simulator()`。 |

### 3.2 指令层关键规则新增

在所有平台的 `INSTRUCTION.md` 中，强制插入以下阻断性逻辑：

- **Rule 7（信息确权门禁）**：执行四维创意发散前，必须先完成“联网检索摘要展示”并等待用户明确回复“已确认/继续”。严禁未展示检索结果直接输出论题。
- **Rule 8（时间窗口交互）**：每次联网检索前，必须向用户展示当前时间窗口（如“近 2 年”），并提供修改入口。
- **Rule 9（降重与去 AI 化优先级）**：`style_normalizer` 的执行优先级高于 `report_generator` 的排版。输出内容若检测到 `forbidden_ai_phrases.json` 中的词汇，必须替换后方可输出。
- **Rule 10（后置交互循环）**：报告输出后，禁止结束对话。必须渲染“深度辅助导航菜单”，并等待用户选择下一步动作（文献精读/实验预研/答辩模拟）。

## 4. I/O 接口与 Schema 升级

为支持多粒度与全流程引导，对去上下文化 I/O 的 JSON Schema 进行升级。

### 4.1 `input_schema.json` 增量字段

```json
{
  "degree": "master | phd",
  "lineage": "string",
  "strategy": "advisor_extension | peer_inheritance | cross_domain | contradiction_driven | all",
  "count": "1-10",
  "output_format": "dialogue | file",
  "output_granularity": "concise | standard | detailed",  // 新增：颗粒度控制
  "inspiration_time_window": "1y | 2y | 3y | 5y",          // 新增：灵感勘探时间窗
  "novelty_time_window": "3y | 5y | 10y"                   // 新增：查重评估时间窗
}
```

### 4.2 `output_schema.json` 增量字段

```json
{
  "status": "success | retry | error",
  "data": {
    "proposals": [
      {
        "title": "string",
        "score": "float",
        "novelty_risk": "low | medium | high",             // 新增：新颖性风险评级
        "novelty_report": "string"                         // 新增：重合度与差异化空档说明
      }
    ],
    "report": "string (Markdown)",
    "high_plagiarism_risk_sections": ["string"]            // 新增：局部高重复风险段落标红提示
  },
  "next_actions": [                                        // 新增：后置引导菜单
    "literature_deep_reading",
    "experiment_design",
    "defense_simulation"
  ],
  "error_message": "string | null"
}
```

## 6. 总结与实施建议

V4.0 架构并未推翻 V2.1 精心设计的三层物理结构与 I/O 去上下文化哲学，而是在其之上植入了**学术常识校验（联网确权）**、**用户交互控制感（粒度/时间）**以及**长期主义价值（后续辅助）**。

通过改进，ThesisMinerSkill 将从一个“听话的文书生成工具”，进化为具备学术审慎态度和科研陪跑能力的智能体，真正契合研究生从开题到实验的全流程心智模型。实施时应优先推进 `search_strategies.json` 与 `style_normalizer.py` 的开发，以最快速度解决时效性与 AI 查重率两大硬伤。
